Element manupulation

nort.createElement( tagname, attributes, [ innerELements .... ])

  • returns: the DOM element created.
  • tagname: <string> HTML tag of the element to create
  • attributes: <object> attributes of the element
  • innerELements: child elements of the new created element. This can be
    • Individual parameters each specifying a DOM element or a NORT component with a render() method.
    • And array of children. This is a alternate way of passing children list.

createElement() is the core of the NORT framework. It is an improved version of document.createElement that can create a DOM element and its child nodes in one function call. Most developers should not need tu use createElement directly and use the HTML5 element generators.

nort.addElement(parent, innerELements)

  • returns: nothing
  • parent: the parent element.
  • innerELements: the new child element or a NORT component with a render() method to append, or an array containing a combination of the two types of objects

addElement() is an improved version of element.appendChild that can handle both DOM element and NORT components in a single way. Its primary use is to be a helper function for createElement()

nort.emptyElement(elt)

  • returns: nothing
  • elt: the parent element.

emptyElement(elt) complely removes the inner content of an element without removing the element itself.

nort.setInnerContent(parent, innerELements)

  • returns: nothing
  • parent: the parent element.
  • innerELements: the new child element or a NORT component with a render() method to append, or an array containing a combination of the two types of objects

    • setInnerContent() replaces the inner contents of an element with a new one. This functionality is especially useful for refreshing contents.

    nort.destroyElement(elt)

    • returns: nothing
    • elt: the element to remove.

    destroyElement(elt) complely removes an element from the DOM.

    HTML5 elements

    NORT includes a full set of generator function for all HTML5 elements. The functions are friendly wrappers of createElement() that make your Javascript UI as readeable as HTML. They eleminate the need for a meta-language, transpiliing, and a complex development toolchain.

    ${tagname}(attributes, [ innerELements .... ])

    {tagname} is the name of the DOM element to create. For example, to create a new DIV element containing a header and some paragraph, the code looks like this: 

        my element = $div( {style: "background-color: silver" },
                    $hl({},"This is the title"),
                    $p({}, "This is the paragraph content")
                        )

    Which is the NORT equivalent for of the following HTML:

        <div style="background-color: silver">
        <h1>This is the title</h1>
        <p>This is the paragraph content</p>
    </div>
